# 深度相机帧数据包

发送 RGB + 深度图到重建引擎。

---

## 请求

```
POST /api/vehicle/frame
Content-Type: application/json
```

```json
{
  "timestamp": 1715808000.456,
  "image": "<base64 JPEG>",
  "depth_map": "<base64 16-bit PNG>"
}
```

| 字段 | 类型 | 格式 | 说明 |
|------|------|------|------|
| timestamp | float | Unix 秒 | 采集时间戳 |
| image | string | Base64 JPEG, BGR8 | RGB 彩色图，约 50-150KB |
| depth_map | string | Base64 16-bit PNG | 深度图，每像素值=深度 mm，约 100-300KB |

频率：根据深度相机帧率，建议 640×480@30fps 或按需降低。

## 深度图编码说明

```
原始深度值: uint16, 单位 mm
传输格式:   16-bit 单通道 PNG（无损压缩）
服务端解码: cv2.imdecode → uint16 numpy (H×W)
转米制:     depth_meter = depth_map / 1000.0
转点云:     针孔模型, X=(u-cx)*Z/fx, Y=(v-cy)*Z/fy
```

## 响应

```json
{"status": "ok", "frame_id": "a1b2c3d4"}
```

## 单帧大小估算

| 组成部分 | 640×480 | 1280×720 |
|----------|---------|----------|
| RGB JPEG | ~50-100KB | ~80-150KB |
| 深度 16bit PNG | ~100-300KB | ~400-800KB |
| **合计** | **~150-400KB** | **~500-950KB** |
