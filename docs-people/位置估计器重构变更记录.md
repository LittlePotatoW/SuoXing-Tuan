# 位置估计器重构 + 前端建模统一 — 变更记录

## 后端 estimation 模块重写

### 删除的文件
- `backend/server/estimation/facade.py` — 旧 PositionEstimator 门面
- `backend/server/estimation/bicycle.py` — BicycleStrategy
- `backend/server/estimation/constant.py` — ConstantStrategy（未被 facade 使用）
- `backend/server/estimation/rgbd_odometry.py` — RgbdOdometryStrategy
- `backend/server/estimation/fusion.py` — FusionStrategy

### 新建的文件
- `backend/server/estimation/base.py` — Position 命名元组 + BaseEstimator ABC（纯接口）
- `backend/server/estimation/bicycle_estimator.py` — BicycleEstimator，自包含自行车运动学
- `backend/server/estimation/constant_estimator.py` — ConstantEstimator，自包含匀速外推
- `backend/server/estimation/rgbd_estimator.py` — RgbdEstimator，自包含 Open3D + actor + 解码
- `backend/server/estimation/fusion_estimator.py` — FusionEstimator，自包含 EKF + Open3D + actor
- `backend/server/estimation/manager.py` — EstimatorManager，路由类，统一管理四个模式
- `backend/server/estimation/__init__.py` — 导出 Position / EstimatorManager / get_manager()

### 修改的文件
- `backend/server/api/routes/vehicle.py`
  - `from server.estimation import PositionEstimator, Position` → `from server.estimation import Position, get_manager`
  - `PositionEstimator.get()` → `get_manager()`
  - `est.update_telemetry(...)` → `mgr.handle_telemetry(...)`
  - `est.update_frame(...)` → `mgr.handle_frame(...)`
  - `reset_estimator`: 15 行手动工厂 + 私有属性赋值 → 6 行 `mgr.create(mode, overrides)`
- `backend/server/engine/engine.py`
  - 两处 `PositionEstimator.get().get_position_at(ts)` → `get_manager().get_position_at(ts)`
- `backend/server/estimation/manager.py`
  - `threading.Lock()` → `threading.RLock()`（修复双重加锁死锁）
  - `switch` → `create`, `shutdown` → `stop`
- `backend/server/api/schemas/vehicle.py`
  - `EstimatorResetRequest` 新增 `fusion_weight: float | None = None`
  - `EstimatorConfigResponse` 新增 `fusion_weight: float = 0.0`，`wheelbase/constant_speed` 加默认值

## 前端改造

### 新建的文件
- `frontend/src/stores/estimation.ts` — 模式共享状态（mode/wheelbase/constantSpeed + shouldSendTelemetry）
- `frontend/src/composables/useModeling.ts` — 统一建模生命周期（Replay/Realtime 共用 start/stop）

### 修改的文件
- `frontend/src/composables/useConnection.ts`
  - 新增 `import { useEstimationStore } from '@/stores/estimation'`
  - 遥测转发加条件：`modelingActive && useEstimationStore().shouldSendTelemetry`
- `frontend/src/views/SettingsView.vue`
  - `estimatorApplied` watch 不再调 `resetEstimator()`，改为存 `localStorage('suoxingtuan_est_config')`
  - `onMounted` 改为优先读 localStorage，无缓存时才从后端读
  - 移除 `resetEstimator` import
- `frontend/src/views/ReplayModeling.vue`
  - 用 `useModeling` 替代重复的 reset/resetEstimator/report/WS 逻辑
  - 遥测循环加条件：`estStore.shouldSendTelemetry`
- `frontend/src/views/RealtimeModeling.vue`
  - 用 `useModeling` 替代重复逻辑
  - **注释掉 polling 代码**：`frameCount/frameThreshold` ref、`statusTimer/fallbackTimer`、`watch(wsConnected)` 降级轮询、`pollStatus/fallbackPoll` 函数
  - 模板：`帧: {{ frameCount }}/{{ frameThreshold }}` → `帧: --/--（待实现）`
  - 注释掉 import：`watch` / `getReconstructionStatus` / `wsConnected`
- `frontend/src/types/api.ts`
  - `EstimatorResetRequest` 新增 `fusion_weight?: number | null`
  - `EstimatorConfigResponse` 新增 `fusion_weight: number`
