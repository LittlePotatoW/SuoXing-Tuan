# 总体设计

我们现在要开发一个项目，前后端两个架构：
- 前端是 Electron 的 Vue 前端(目前先实现使用浏览器就行)
- 后端是 Python Server

## 项目目标

接收小车发来的激光雷达数据、图像数据、速度、转向角度，进行三维重建和图像模型识别（框），然后将识别结果（框的中心）在三维重建的场景中标注

## 项目层次结构
从外层向内层依次是（数字越大越内层，可能有并列）：
- 网络接发层（前端，最外层，0）：
远程连接（http，websocket或者局域网连接）接收数据，或者发送数据
- 数据加载层（前端，最外层，0）：
部分模式，直接加载本地数据进行处理
- 显示层（前端，最外层，0）：
接收最终数据，显示，比如渲染三维场景，比如渲染单张图像推理结果
- 数据保存层（前端，0.5）:
有一些功能，可能涉及将网络接收到的数据保存到本地
也可能涉及到将重建完的结果保存
- 包装与解包层（前端，1）：
将接收到的数据包（网络接发层）转换成前端使用的统一格式。
- 渲染引擎（前端，1）：
将后端传上来的数据，进行一些处理，最后在显示层显示出来。
将推送上来的数据包封装成统一格式，进行网络请求（网络接发层）
- 数据预处理层（前端，1.5）：
主要是预处理激光雷达点云和图像数据，进行翻转，对齐，裁剪等操作
- 前端API层（前端，2）：
封装了与后端交互的api接口，
前端其他层，通过调用这些函数，来向后端请求或发送数据。
- 后端API层（后端，3）：
封装了与前端交互的api接口，后端其他层，通过调用这些函数，来向前端请求或发送数据。
- “重建引擎”（后端，4，详见docs-people\promote\详细部分设计\“重建引擎”.promote.md）：
不定时得到前端推送的“帧数据”（对齐的激光雷达和图像数据，带时间戳），然后向位置估计器请求位置，积累帧数，帧数达到一定值时，触发重建，并将重建结果发到前端
- “位置估计器”（后端，4.5，详见docs-people\promote\详细部分设计\“位置估计器”.promote.md）：
维护小车的位置，前端时不时推送小车的速度和转向数据（带时间戳），后端持续维护位置，后端的重建引擎随时可以请求得到小车的位置。

## 前端功能界面设计
所有界面，左边有导航栏，左上角是logo和“SuoXing-Tuan”字样，上面是一个状态栏，显示连接状态
- 界面一：
主界面，显示小车实时图像（摄像头图像），激光雷达点云灰度图，以及小车位置信息（速度，转向角度），还有一张地图（先预留，以后可能有地图接口和GPS数据），以及一个按钮，点击后才开始轮询数据。所有数据均不要求视频流，简单是https慢慢获取。
- 界面二：
实时建模界面，启动后，持续获取小车速度，转向，点云和图像信息，持续yolo模型检测与，渲染建模。
这个界面右边有个配置栏，可以配置一些选项
- 界面三：
回放建模界面，可以使用本地数据进行建模，类似上面实时建模，走一样的管线，只是数据源由“网络接发层”变成了“数据加载层”
- 界面四：
详细信息界面，可以看到“实时建模界面”检测到的缺陷列表和缺陷图像（带标注框）
- 界面五：
静态图像检测界面，拖动图片上传，点击检测后显示推理结果
- 界面六：
配置界面，配置中转服务器啥的ip与端口，配置后端的ip与端口，配置建模参数，比如“全量重建”或者“增量重建”，多少帧触发重建
