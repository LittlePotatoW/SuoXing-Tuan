## 与论文方案对比

| 步骤 | 论文方案 | SuoXing-Tuan 方案 |
|------|----------|-------------------|
| 数据采集 | MTI-300 (LSC + MLS) | 巡线小车 (LiDAR + 工业相机) |
| 缺陷检测 | YOLOv8-seg (2D) | YOLO (裂缝检测, `/api/inference`) |
| 点云→2D | 圆柱展开 + 方格划分 | 不需要（直接在 3D 做） |
| 图像对齐 | SIFT+FLANN 特征匹配 + 单应矩阵 | **LiDAR-Camera 外参标定 + 内参投影** |
| 颜色映射 | 颜色空间融合（缺陷→红色） | **3D 点反投影采样真实像素颜色** |
| 3D 重建 | 网格三角化 + 圆柱卷曲 | **Poisson 表面重建** |
| 颜色传递 | 无（仅红色标注缺陷） | **KDTree K近邻距离加权** |
| 定位 | 不需要（2D 平面配准） | **自行车模型航迹推算** |
| 可视化 | Unity OBJ | **Three.js Web 实时渲染** |

## 关键差异

```
论文:  2D 降维做配准 ──→ 图像特征对齐 ──→ 缺陷标注叠回 3D
        (需要螺栓孔等纹理特征)

我们:  3D 原生做融合 ──→ 几何级精确投影 ──→ 全彩色纹理渲染
        (需要外参+内参标定，不依赖表面纹理)
```
