# 多类别缺陷检测升级方案

## 一、现状

| 项目 | 当前 |
|------|------|
| 模型 | YOLOv8n (3.2M 参数) |
| 数据集 | DeepCrack (单类别: crack) |
| 图片数 | 300 train / 237 val |
| 标注数 | ~1,910 bbox |
| mAP@0.5 | ~0.29 |
| 代码位置 | `model-training/yolo-training/` |

## 二、数据集推荐: CODEBRIM

**CODEBRIM** (COncrete DEfect BRidge IMage Dataset) — CVPR 2019, 歌德大学

| 项目 | 详情 |
|------|------|
| 图片 | 1,590 张高分辨率桥梁图片 (30座不同桥梁) |
| 标注 | 5,354 个 bbox |
| 类别 | **6 类**: crack(裂缝), spallation(剥落), rebar(露筋), efflorescence(泛碱), corrosion(腐蚀), background(无缺陷) |
| 大小 | ~500MB |
| 格式 | Pascal VOC XML |
| 下载 | https://zenodo.org/records/2620293 |
| 许可 | 非商业/教育用途 |

### 为什么选 CODEBRIM

- 混凝土桥梁场景，与当前隧道裂缝方向一致
- 6 个类别覆盖了最常见的混凝土病害类型
- 学术引用广泛，质量有保障

### 其他备选

| 数据集 | 大小 | 类别数 | 格式 | 说明 |
|--------|------|--------|------|------|
| DamSegment | ~300MB | 2 (crack, spalling) | 原生 YOLO | 最简单，即下即用 |
| NEU-DET | 27MB | 6 (钢材缺陷) | 原生 YOLO | 最小，但是钢材表面不是混凝土 |
| HRCDS | 195MB | 4 | 分割 mask | 需要 mask→bbox 转换 |
| S2DS | ~250MB | 7 | 分割 mask | Google Drive，国内可能不稳定 |

## 三、模型推荐: YOLOv11s

| 模型 | 参数量 | 说明 |
|------|--------|------|
| **YOLOv11s** (推荐) | ~9.4M | 最新、缺陷检测 benchmark 最优 |
| YOLOv8s (备选) | ~11.2M | 更成熟，社区资源最多 |
| YOLOv11n | ~2.6M | 最轻量，但多类别可能容量不够 |

### 选 YOLOv11s 的理由

- Transformer-hybrid 检测头对小缺陷更敏感（比 YOLOv8 召回率高约 8.7%）
- NEU-DET 钢材缺陷上比 YOLOv8 准确率高约 70%
- 与 YOLOv8 共用 `ultralytics` API
- s 版本在精度和速度之间平衡最好

## 五、实施步骤

### Step 1: 创建目录 & 下载数据

- 新建 `model-training/multi-class-training/`
- 从 https://zenodo.org/records/2620293 下载 CODEBRIM
- 解压到 `datasets/CODEBRIM/`，确保有 `Images/` 和 `Annotations/` 子目录

### Step 2: 创建 `dataset.py`

- 从 `yolo-training/dataset.py` 复用核心函数:
  - `convert_voc_to_yolo()` — VOC XML → YOLO txt
  - `generate_data_yaml()` — 生成 yaml 配置
- 新增 `prepare_codebrim()` — 串联整个流程: 读 CODEBRIM → 80/20 分割 → 转换 → 生成 yaml
- 注意: 需要先检查 XML 里的实际类名，调整 class_map 映射

### Step 3: 创建 `train.py`

- 从 `yolo-training/train.ipynb` 提取训练逻辑为独立脚本
- 模型: `yolo11s.pt`
- 超参数沿用现有配置: 100 epochs, batch 16, imgsz 640, lr0=0.01 等
- 增加 per-class AP 评估输出

### Step 4: 创建 `requirements.txt`

```
ultralytics>=8.3.0
torch>=2.0.0
numpy>=1.26.0
opencv-python>=4.9.0
matplotlib>=3.7.0
tqdm>=4.65.0
PyYAML>=6.0
scikit-learn>=1.3.0
```

### Step 5: 训练 & 验证

- 运行 `prepare_codebrim()` 做数据转换
- 运行 `train.py` 开始训练
- 训练完成后，`best.pt` 复制到 `backend/models/` 替换旧模型
- 后端/前端均无需改代码（已原生支持多类别）

## 七、风险点

1. **CODEBRIM XML 类名不确定** — 下载后需先抽查几个 XML 确认 `<name>` 实际值，调整 `class_map`
2. **类别不平衡** — background 类样本可能远超缺陷类，训练时可考虑过滤 background 或加类别权重
3. **高分辨率图片** — CODEBRIM 原图最高 6000x4000，YOLO 训练会自动 resize 到 640，小缺陷可能丢失，可考虑提高 imgsz 到 1280
4. **Zenodo 下载速度** — 国内可能较慢，如无法下载可换用 DamSegment (Mendeley, 原生 YOLO 格式)
