# 前端双网络模式设计

## 两种模式

| 模式 | 架构 | 适用场景 |
|------|------|----------|
| 局域网直连 (LAN) | 前端 ←→ 小车后端 | 本地调试、室内同一网络 |
| 服务器中转 (Server) | 小车 → 云服务器 ← 前端 | 远程监控、小车室外运行 |

## 局域网模式

```
┌──────────┐   HTTP (POST /telemetry, /frame)
│ 前端     │ ←──────────────────────────
│ (浏览器) │ ──────────────────────────→
└──────────┘   HTTP (GET /position, /status, /result)
        │
        │ WebSocket (/ws)
        │   ← 重建结果推送 / 状态更新
        ▼
┌──────────┐
│ 小车后端  │  (Python Server, 跑在小车上)
│ :8000    │
└──────────┘
```

- 小车跑完整后端（API + 估计器 + 引擎 + 检测）
- 前端直连小车 IP
- 零外部依赖，最低延迟

## 服务器模式

```
┌──────────┐                          ┌──────────┐
│ 前端     │   HTTP + WebSocket       │ 云服务器  │   HTTP + WebSocket    ┌──────────┐
│ (浏览器) │ ←─────────────────────→ │ :8000    │ ←───────────────────→ │ 小车后端  │
└──────────┘                          └──────────┘                       └──────────┘
```

- 云服务器上跑后端（或作为代理转发）
- 小车推送数据到服务器，前端从服务器拉数据
- 小车和前端不需要在同一网络

## 前端配置项

```typescript
interface NetworkConfig {
  mode: 'lan' | 'server'

  // 局域网模式
  lan: {
    host: string       // 小车 IP，如 '192.168.1.100'
    port: number       // 后端端口，默认 8000
  }

  // 服务器模式
  server: {
    host: string       // 服务器地址，如 'your-server.com'
    port: number       // 服务器端口，默认 8000
  }
}
```

## 前端需要实现的模块

### 1. `src/config/defaults.ts` — 默认配置
- 导出 `LAN_CONFIG`（默认小车 IP + 端口）
- 导出 `SERVER_CONFIG`（默认服务器地址 + 端口）
- 导出 `NETWORK_MODE` 枚举

### 2. `src/stores/settings.ts` — 用户当前选择的模式
- `networkMode: 'lan' | 'server'` 状态
- 切换模式时更新 HTTP client 的 baseURL 和 WebSocket 地址

### 3. `src/network/http-client.ts` — 动态 baseURL
- 不写死地址，初始化时从 settings store 读取当前模式对应地址
- `setBaseURL(url)` 方法支持运行时切换

### 4. `src/network/websocket-client.ts` — 动态 WS 地址
- 同理，连接地址从 settings store 读取

### 5. `src/components/layout/AppStatusBar.vue` — 状态栏显示
- 显示当前模式图标（LAN / Server）
- 显示连接状态

### 6. 界面六（SettingsView.vue）— 配置界面
- 选择网络模式（单选框 LAN / Server）
- 填入对应的 IP 和端口

## 后端是否需要改动？

**不需要。** 两种模式后端接口完全一致，只是连接方不同：

- 局域网模式：前端直接调 `192.168.1.100:8000/api/vehicle/telemetry`
- 服务器模式：前端调 `your-server.com:8000/api/vehicle/telemetry`，服务器可能做代理转发或小车直接推数据到服务器

后端一直暴露同一套 API，换 IP 就行。
