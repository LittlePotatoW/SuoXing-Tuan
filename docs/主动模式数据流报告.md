# 主动模式数据流报告（当前代码）

## 1. 三种子模式

| 子模式 | 数据从哪来 | 怎么播放 |
|--------|-----------|---------|
| **中继 Relay** | TranspondServer WS `/stream?mode=all`（优先）+ HTTP 轮询（WS 断开时 fallback） | 实时流式，逐帧转发后端 |
| **回放 Replay** | TranspondServer `GET /location?limit=N` + `GET /sensor?limit=N` 一次拉到内存 | 按时间戳间隔逐帧播放 / 一键渲染 |
| **手动 Manual** | 本地 `data_captures/<name>/`（通过后端 debug API） | 逐帧播放 / 一键渲染；fused 路径跳过融合 |

三种模式最终走相同的后端管线：`/api/realtime/feed/location` 和 `/api/realtime/feed/detection`（手动 fused 模式走 `/api/realtime/feed/fused`）。

---

## 2. 重建需要的两路数据

### 定位数据 → 状态估计
`POST /api/realtime/feed/location` → `DataFusionManager.feed_location_data()`:
- 缓存 `camera[].camera_pose`（外参池）
- 喂入 `StateEstimator.update_kinematics()`（自行车模型积分）

### 检测数据 → 融合 + 重建
`POST /api/realtime/feed/detection` → `DataFusionManager.process_detection()`:
1. 从 StateEstimator 查 `timestamp_ns` 对应的世界位姿
2. 用缓存外参覆盖 camera_pose
3. 组装 SensorFrame → DataFusion（坐标系变换） → ReconstructionEngine
4. 每 10 帧 Poisson 重建 → WS 推送 `rebuild_complete`

**重建核心输入**: 点云 + 图像 + 姿态，三者时间对齐。

---

## 3. 存储格式

保存模式 (`saveOn=true`) 下，数据写入 `data_captures/<saveName>/`：

| 类型 | 目录 | 保存时机 | 保存什么 |
|------|------|----------|---------|
| location | `location/00001.json` | `forwardLocation()` 成功时 | Transpond 原始 loc 数据 |
| sensor | `sensor/00001.json` | `forwardSensor()` 成功时 | 原始 sensor frame |
| fusion | `fusion/00001.json` | WS 收到 `rebuild_complete` 时 | mesh + cracks + camera_trail |

**注意**: 代码中 `forwardSensor()` 第 383 行调用了 `doSave('fused', j.final)`，但后端 `debug/routes.py` 目前**没有** `/save/fused` 端点。这个保存会静默失败（被 catch 捕获）。

---

## 4. 手动模式加载

`manualLoad()` 并行请求三个端点：
```
GET /api/debug/sessions/<name>/sensor  → manualSensorFrames
GET /api/debug/sessions/<name>/location → manualLocFrames  
```

**决策逻辑**（`manualHasFused` / `manualCanRawRebuild`）：

- 有 sensor 且（有 loc 或匀速直线开启） → `manualReplayMode = 'raw'`, `dataStatus = 'full'`
- 否则 → `dataStatus = 'error'`

**重要**: ManualPanel.vue 的 radio 值用 `'fusion'`，但 RealtimeView.vue 的 `manualReplayMode` 用 `'fused'`，**两者不匹配**，radio 选中状态不会正确显示。

---

## 5. 后端 debug API（实际存在的端点）

### 保存
| 端点 | 写入目录 |
|------|---------|
| `POST /api/debug/save/location?session=X` | `data_captures/X/location/` |
| `POST /api/debug/save/sensor?session=X` | `data_captures/X/sensor/` |
| `POST /api/debug/save/fusion?session=X` | `data_captures/X/fusion/` |

### 读取
| 端点 | 返回 |
|------|------|
| `GET /api/debug/sessions` | 所有 session 名列表 |
| `GET /api/debug/sessions/{name}/info` | location/sensor/fusion 文件数 |
| `GET /api/debug/sessions/{name}/location` | `{"frames": [...]}` |
| `GET /api/debug/sessions/{name}/sensor` | `{"frames": [...]}` |
| `GET /api/debug/sessions/{name}/fusion` | `{"frames": [...]}` |

**没有** `/save/fused` 和 `/sessions/{name}/fused` 端点。

---

## 6. `/api/realtime/feed/fused` 端点（手动 fused 回放用）

`POST /api/realtime/feed/fused`（`fusion_router.py` 第 127 行）:
- 从 `config.yaml` 补全缺失的 `camera_pose`
- 构建 SensorFrame → DataFusion → ReconstructionEngine
- 跳过 `feed_location` + `feed_detection` 两个中间层

---

## 7. 匀速直线模式

开启后：
- **定位**：不发送真实 loc，合成 `{velocity: speed, steering_angle: 0}` 发到 `/api/realtime/feed/location`
- **检测**：`forwardSensor()` 覆盖 `car_position = {x: speed*dt, y:0, z:0}`
- 跨所有子模式生效

---

## 8. 发现的潜在问题

1. `manualReplayMode` 值用 `'fused'`，ManualPanel radio value 用 `'fusion'`，不匹配
2. `doSave('fused', ...)` 调用了不存在的后端端点
3. `manualLoad()` 没有请求 `/sessions/X/fused`（该端点也不存在），`manualHasFused` 始终为 false
