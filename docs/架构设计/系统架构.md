# 系统架构

## 子系统清单

| 子系统 | 语言/框架 | 端口 | 位置 | 说明 |
|--------|-----------|------|------|------|
| 控制端 (Backend) | Python FastAPI | 8000 | `backend/` | AI 推理、数据融合、三维重建 |
| 前端 (Frontend) | Vue 3 + Three.js | Vite dev | `frontend/` | 检测/重建/实时融合/监控/配置 |
| 数据中继 (Transpond) | C++17 + cpp-httplib | 8001 | `OTHER_END/Transpond_Server/` | 硬件数据接收、缓存与转发 |
| 控制中转 (Control) | C++17 + cpp-httplib | 8080 | `OTHER_END/Control_Server/` | 手机-小车 WebSocket 房间中转 |
| 车端采集 (Car) | C++17 | — | `OTHER_END/Car_detection_end/` | LiDAR + 相机数据采集与上传 |
| 模型训练 | Python PyTorch | — | `model-training/` | YOLO 裂缝检测模型训练 |
| 模拟器 | Python | 8765 | `test_end/` | 离线测试数据生成与发送 |
| 启动脚本 | Python | — | `script/` | 前后端与模拟器启停 |

## 整体架构

```
车端采集 (C++)                        控制端 (FastAPI)                   前端 (Vue3+Three.js)
┌──────────────┐    POST /location    ┌──────────────────────────┐    ┌──────────────────────┐
│ LiDAR 点云    │───→ POST /frames ──→│ 数据中继站 TranspondServer │───→│ RealtimeView         │
│ USB 相机      │                     │ (:8001, 缓存+转发)        │    │ (主动模式拉取)        │
└──────────────┘                     └──────────┬───────────────┘    └──────────────────────┘
                                                 │
                                                 ▼
                                        ┌──────────────────────────────┐
                                        │ inference/ (YOLO AI 推理)     │
                                        │ fusion/ (数据融合+颜色采样)    │──→ WS 推送 → 前端 3D 查看器
                                        │ reconstruction/ (三维重建)     │
                                        │ state_estimation/ (航迹推算)   │
                                        │ realtime/ (实时调度层)         │
                                        │ loader/ (离线回放)             │
                                        └──────────────────────────────┘
```

```
控制中转 (C++)                         手机 ↔ 小车 远程控制
┌──────────────────────┐
│ Control_Server       │    WS /phone  ← 手机连接（控制指令 + 遥测接收）
│ (:8080, 房间中转)     │    WS /robot  ← 小车连接（遥测上报 + 指令转发）
└──────────────────────┘
```

## C++ 服务

### 数据中继站 (Transpond_Server)

- 接收车端 POST 的定位数据和传感器帧，存入环形缓冲区
- 提供 REST 查询接口 + WebSocket 实时推送
- 支持 `--port`、`--max-loc`、`--max-sensor` 命令行参数配置

### 控制中转 (Control_Server)

- WebSocket 房间模型：手机 (`/phone`) 和小车 (`/robot`) 按房间 ID 配对
- 小车上报遥测 (tele/loc)，手机下发控制指令 (ctrl/loc_cfg)
- 每房间独立环形缓冲区存储遥测数据

### 车端采集 (Car_detection_end)

- 读取 Livox Mid-40 LiDAR + USB 相机数据
- 通过 HTTP POST 发送到 Transpond_Server
