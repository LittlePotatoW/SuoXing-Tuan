# 数据保存与回放格式

## 存储结构

一个录制会话对应一个目录：

```
session_20240718_120000/
├── manifest.json          # 元信息 + 遥测时间线
├── frames/
│   ├── 00001.jpg          # RGB 图像 (JPEG)
│   ├── 00001.png          # 深度图 (16-bit PNG)
│   ├── 00002.jpg
│   ├── 00002.png
│   └── ...
```

- 图像按序号命名，不嵌时间戳——时间戳在 manifest 里
- 图像存原始文件（不用 base64），节省空间，回放时直接 FileReader 读取

## manifest.json

```jsonc
{
  "version": 1,
  "start_time": 1715808000.000,       // 录制开始 Unix 时间戳
  "end_time":   1715808060.000,       // 录制结束
  "frame_count": 1800,                // 帧总数
  "telemetry_interval_ms": 100,       // 遥测采样间隔

  // 遥测时间线: 按采集顺序排列, 每个元素是一条遥测
  "telemetry": [
    { "ts": 0.000, "speed": 0.0, "steering": 0.0 },
    { "ts": 0.100, "speed": 0.5, "steering": 2.0 },
    { "ts": 0.200, "speed": 1.0, "steering": 5.0 }
    // ...
  ],

  // 帧索引: 每帧对应一组图像文件
  "frames": [
    { "id": 1, "ts": 0.050, "image": "frames/00001.jpg", "depth": "frames/00001.png" },
    { "id": 2, "ts": 0.150, "image": "frames/00002.jpg", "depth": "frames/00002.png" }
    // ...
  ]
}
```

## 字段说明

### 遥测

| 字段 | 类型 | 单位 | 说明 |
|------|------|------|------|
| ts | float | 秒 | 相对录制开始的时间偏移 |
| speed | float | m/s | 速度，>=0 |
| steering | float | 度 | 车轮转角，正=左转 |

- ts 用相对时间而非绝对时间戳，方便回放时从 0 开始
- 遥测和帧数据在 manifest 中都存，按各自频率

### 帧

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 帧序号，自增 |
| ts | float | 相对录制开始的时间偏移（秒） |
| image | string | RGB JPEG 文件相对路径 |
| depth | string | 深度 16-bit PNG 文件相对路径 |

## 回放流程

```
1. 用户选择 session 目录
   → 读取 manifest.json
   → 校验 version, frame_count

2. 回放引擎:
   → 从 ts=0 开始
   → 按时间轴同时推进遥测和帧数据
   → 遥测: 按 telemetry_interval_ms 间隔取数 → postTelemetry()
   → 帧: 到达帧的 ts 时 → 读 JPEG/PNG 文件 → base64 编码 → postFrame()
   → 支持暂停/快进/慢放

3. 回放速度:
   → 1x: 按原始间隔
   → 2x: 间隔减半
   → 0.5x: 间隔加倍
```

## 保存流程

```
1. 用户点击"开始录制"
   → 创建 session_YYYYMMDD_HHMMSS/ 目录
   → 创建 frames/ 子目录
   → 记录 start_time = now()

2. 每收到一条遥测:
   → 追加到内存 telemetry 数组 { ts: now - start_time, speed, steering }

3. 每收到一帧:
   → base64 解码 → 存为 frames/00001.jpg + frames/00001.png
   → 追加到内存 frames 数组 { id, ts: now - start_time, image: 相对路径, depth: 相对路径 }

4. 用户点击"停止录制":
   → 写入 manifest.json
   → 打包整个 session 目录为 zip（可选）
```

## 与各层的关系

```
录制时:
  Layer 0 (网络接发层) 收到数据
    → Layer 1 (包装与解包) 解析
      → Layer 0.5 (数据保存) 写入文件

回放时:
  Layer 0 (数据加载) 读取文件
    → Layer 1 (包装与解包) 解析
      → Layer 2 (API) 发送到后端
```
