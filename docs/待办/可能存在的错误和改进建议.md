# 可能存在的错误和改进建议

> 4 个并行代理全面审查, 覆盖后端重建管线 / API 路由 / 前端渲染 / 前端视图与状态管理, 共发现 ~90 个问题。

---

## 一、Critical — 安全漏洞

### 1. session.py 路径穿越漏洞（写入）

- **文件:** `backend/server/api/routes/session.py:24-25`
- **问题:** `body.name` 直接拼入 `SESSION_DIR / body.name`, 未过滤 `../`。攻击者可写任意文件
- **修复副作用:** 校验 name 不含 `/` `\` `..` 即可, 正常 session 名不受影响

### 2. session.py 路径穿越漏洞（读取）

- **文件:** `backend/server/api/routes/session.py:59-70`
- **问题:** `GET /files/{session_name}/{file_path:path}` 的 `:path` 转换器允许 `../`, 可读任意文件
- **修复副作用:** resolve 后检查路径前缀在 SESSION_DIR 内

### 3. report.py 路径穿越漏洞

- **文件:** `backend/server/api/routes/report.py:20-44`
- **问题:** 与 session.py 相同的 `../` 问题, 读/写均可被利用
- **修复副作用:** 同 session.py 方案

## 二、Critical — 几何计算错误

### 4. RGB-D 里程计坐标变换错误（facade.py）

- **文件:** `backend/server/estimation/facade.py:140-141`
- **问题:** `update_frame` 将相机系增量 (dx, dz) 直接当世界系增量累加, 未按 heading 旋转。相机 Z 向前映射为世界 X, 但代码处理反了
- **副作用:** 修正后 rgbd/fusion 模式的轨迹几何正确, 但历史轨迹会变

### 5. EKF 观测向量坐标变换错误（fusion.py）

- **文件:** `backend/server/estimation/fusion.py:68`
- **问题:** `z = np.array([x + dx, y + dz, heading + dh])` — 同样的相机→世界旋转变换错误
- **副作用:** 同 #4, fusion 模式修正后坐标正确

### 6. 法线全部朝向原点（surface.py）

- **文件:** `backend/server/reconstruction/surface.py:78`
- **问题:** `orient_normals_towards_camera_location([0.0, 0.0, 0.0])` 把所有法线定向到世界原点, 远离原点的帧法线方向错误, Poisson 产生翻转面/空洞
- **副作用:** 修正后 mesh 质量明显提升。需要用各帧的实际相机位置替代原点

## 三、High — 数据正确性

### 7. 顶点过滤后面索引未重映射（point-cloud.ts）

- **文件:** `frontend/src/services/renderer/point-cloud.ts:31-61`
- **问题:** NaN 顶点被过滤后 `verts` 数组变短, 但 `faces` 仍是原始索引。引用被删顶点的面会读到错误顶点或越界 → 花屏
- **副作用:** 需要建旧索引→新索引映射并重写 faces

### 8. useReconstructionWS disposed 标志阻止重连

- **文件:** `frontend/src/composables/useReconstructionWS.ts:41,90`
- **问题:** `disconnect()` 设 `disposed=true`, `connect()` 检查 `disposed` 直接 return。停止后无法重新开始建模
- **副作用:** 用 `connecting` 替代 `disposed` 或在 connect 时重置

### 9. TypeScript 与 Pydantic 字段名不匹配

- **文件:** `frontend/src/types/api.ts` vs `backend/common/schemas.py`
- **问题:** `bbox`(Python) vs `bbox_2d`(TS), `velocity` vs `speed`, `crack_type` vs `class_name` — 前后端字段名不一致, 运行时值为 undefined
- **副作用:** 统一命名约定, 两边都需改

### 10. RealtimeModeling 轮询退化代码无效

- **文件:** `frontend/src/views/RealtimeModeling.vue:93-101`
- **问题:** watch 缺少 `{ immediate: true }`, 初始值为 false 时不触发, WebSocket 断开后备用轮询永远不会启动
- **副作用:** 加 `{ immediate: true }`

### 11. SettingsView 每次按键触发 store 写入

- **文件:** `frontend/src/views/SettingsView.vue:122-130`
- **问题:** deep watch 在每次按键时调 `applyBackendURL()`, 输入中间态(如端口未输入完)产生非法 URL
- **副作用:** 加防抖 300ms 或改为失焦时提交

### 12. StaticDetection Blob URL 泄漏

- **文件:** `frontend/src/views/StaticDetection.vue:79`
- **问题:** `URL.createObjectURL` 创建的 blob URL 在组件卸载时未 revoke, 内存泄漏
- **副作用:** `onUnmounted(() => URL.revokeObjectURL(previewUrl.value))`

### 13. output 路径与 static mount 不一致

- **文件:** `backend/server/main.py:55-57` + `engine.py:291`
- **问题:** engine 用 CWD 相对路径写 `output/`, 但 FastAPI `/output` 挂载 `backend/output/`。从非 backend 目录启动时, 文件写入和 HTTP 服务不在同一目录 → 404
- **副作用:** 统一用项目根绝对路径

### 14. 重建阻塞线程池

- **文件:** `backend/server/engine/engine.py:168-286`
- **问题:** `_trigger` 同步执行所有重型操作 (YOLO + Poisson) 在线程池线程中, 可能阻塞请求处理
- **副作用:** 移到独立后台线程

### 15. asyncio.create_task 无错误处理

- **文件:** `backend/server/api/routes/vehicle.py:84`
- **问题:** WebSocket 广播 task 未加 done callback, 异常被静默吞掉
- **副作用:** 加 `add_done_callback` 捕获日志

## 五、Medium — 性能

### 19. Python dict 降采样回退极慢

- **文件:** `fusion.py:73-91`
- **问题:** 百万级点 Python for 循环遍历, 比 Open3D 慢 50-100x
- **副作用:** 用 numpy vectorize 优化, 或确保 Open3D 总是可用

### 20. 双重体素降采样

- **文件:** `engine.py:218` + `surface.py:62`
- **问题:** fuse() 降采样一次, surface.py 又降采样一次, 同一数据被处理两遍
- **副作用:** 表面重建开启时 fuse 跳过降采样, 速度 ~2x

### 21. O(n) pop(0) 在轨迹列表

- **文件:** `facade.py:205`
- **问题:** 每次 pop(0) 是 O(n), 10000 条轨迹每次更新都做全量移位
- **副作用:** 换 `collections.deque(maxlen=10000)`, O(1)

### 22. get_position_at 每次重建时间戳列表

- **文件:** `facade.py:168`
- **问题:** 每次调用重新建 timestamps 列表, O(n) 开销
- **副作用:** 维护独立时间戳列表或用 attrgetter

### 23. ASCII PLY Python 循环写入

- **文件:** `engine.py:327-328`
- **问题:** `for i in range(n): f.write(...)` 逐点写, 50 万点需数秒
- **副作用:** `np.savetxt` 替代, 10-100x

### 24. 文件名秒级精度冲突

- **文件:** `engine.py:293` + `surface.py:93`
- **问题:** `time.time():.0f` 仅到秒, 同秒内多次重建会覆盖文件
- **副作用:** 用毫秒精度或 UUID

## 九、设计改进建议（非 Bug）

### 架构

1. **Surface 重建返回类型不一致**: `reconstruct_surface` 标注 `-> str | None` 实际返回 dict
2. **缺少健康检查端点**: 无 `GET /health`, 容器化部署困难
3. **请求体无大小限制**: 无上限, 大 base64 或 manifest 可耗尽内存
4. **错误响应格式不一致**: 有的用 HTTPException, 有的返回 200 空数组, 有的返回 sentinel 值
5. **增量模式点云无界增长**: 遍历面积大时内存无上限
6. **启动时无配置校验**: 相机内参为 0 应启动即报错, 而非运行时静默失败

### 技术栈

- `collections.namedtuple` 可换 `typing.NamedTuple`
- 前端 yaml 配置文件解析每次 import 都重新读, 可缓存
- `Date.now()` 不适合高精度计时, 考虑 `performance.now()`

### 性能优化方向

- Engine push_frame 后的 _trigger 可延迟批量触发（debounce 100ms）
- PLY 文件传给前端可 gzip（文本 PLY 压缩比极高）
- 点云渲染可加 LOD（远处用稀疏采样）
