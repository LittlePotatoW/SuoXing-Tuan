# 局域网连接协议

## 连接方式

小车作为 WebSocket 服务端，前端作为客户端主动连接。

```
前端 (浏览器)                    小车 (Python/WS Server)
───────────                     ──────────────────────
  WS connect ─────────────────→  ws://192.168.1.100:8002
  WS connect ─────────────────→  ws://192.168.1.100:8001  (遥测, 本文档不涉及)
```

帧数据端口 `8002`，走 WebSocket。

## 连接流程

```
1. 前端读取 config.yaml → 获取 LAN 地址和端口
2. 前端 new WebSocket("ws://{ip}:{port}")
3. 小车 WS 服务端 accept 连接
4. 小车开始推送帧数据
5. 前端 onMessage 接收 → 转给后端 API
```

心跳：小车每 30s 发 `ping`，前端回 `pong`。前端 5s 内收不到任何消息则断线重连。

## 消息格式

所有消息为 JSON 字符串，`type` 字段区分消息类型。

### 小车 → 前端

**帧数据**（主要消息，每帧一条）

```jsonc
{
  "type": "frame",
  "timestamp": 1715808000.456,   // Unix 秒
  "image": "<base64 JPEG>",      // RGB 图像, BGR8 解码, 约 50-150KB
  "depth_map": "<base64 PNG>"    // 16-bit 单通道 PNG, 像素值=深度mm, 约 100-300KB
}
```

发送频率：按相机帧率，建议 640×480@30fps 或根据带宽调整。

**心跳响应**

```jsonc
{ "type": "pong" }
```

**状态通知**（可选）

```jsonc
{ "type": "status", "message": "camera_ready" }
```

### 前端 → 小车

**控制指令**（预留，pack.ts 待实现）

```jsonc
{ "type": "control", "command": "set_exposure", "value": 500 }
{ "type": "control", "command": "set_gain", "value": 10 }
{ "type": "control", "command": "set_resolution", "width": 640, "height": 480 }
```

**心跳**

```jsonc
{ "type": "ping" }
```

## 帧数据字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 固定 `"frame"` |
| timestamp | float | 是 | Unix 时间戳（秒），采集时刻 |
| image | string | 是 | Base64 编码的 JPEG，BGR8 色彩 |
| depth_map | string | 是 | Base64 编码的 16-bit PNG，像素值=深度 mm |

## 深度图解码流程（前端收到后）

```
1. base64 解码 → PNG 二进制
2. cv2.imdecode(..., cv2.IMREAD_UNCHANGED) → uint16 numpy (H×W)
3. 原始值 / 1000 → 米制深度
```

前端不需要做深度转点云——这是后端 Layer 3.5 的工作。前端只负责把 `image` 和 `depth_map` 原样通过 `POST /api/vehicle/frame` 转发给后端。

## 带宽估算

| 分辨率 | RGB JPEG | 深度 16-bit PNG | 单帧合计 | 3 Mbps 下最大帧率 |
|------|------|------|------|------|
| 640×480 | ~80KB | ~200KB | ~280KB | ~1.3 fps |
| 320×240 | ~20KB | ~50KB | ~70KB | ~5 fps |

3 Mbps 共享带宽下建议 320×240 @ 2-3 fps，或通过降低 JPEG 质量压缩。

## 断线处理

1. 前端检测 WS `onclose` → 状态栏亮红
2. 指数退避重连：1s → 2s → 4s → 8s → 最大 30s
3. 重连成功后恢复数据流
4. 断线期间不缓存数据（帧数据实时性优先，丢失的帧不可恢复）
