# Docs — 项目文档

## 这个文件夹是干啥的？

放项目的设计文档，包括：
- 系统架构设计（整体怎么搭的）
- 项目结构说明（文件怎么组织的）
- 接口文档（前后端怎么通信）
- 某个功能的设计方案（比如"怎么实现模型热切换"）

## 现有文档

| 文件 | 内容 |
|------|------|
| `ARCHITECTURE.md` | 系统架构设计——从硬件到前端的完整数据流，模块职责划分 |
| `PROJECT_STRUCTURE.md` | 项目结构——每个文件夹放什么、有什么用 |
| `README.md` | 本目录说明 |

## 文档风格

- 面向新手，不拽术语
- 有图有例子
- 主要由 AI 辅助生成，人工确认
